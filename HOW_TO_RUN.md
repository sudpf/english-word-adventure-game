# 如何运行游戏

## 🎮 推荐方式：使用 HTTP 服务器（最佳体验）

### 方法 1：Python HTTP 服务器（已配置）
```bash
cd english-word-adventure-game
python3 -m http.server 8080
```
然后在浏览器访问：**http://localhost:8080**

### 方法 2：Node.js HTTP 服务器
```bash
npx http-server english-word-adventure-game -p 8080
```

## 💡 直接打开文件方式（可能有限制）

### 为什么推荐使用 HTTP 服务器？

虽然这个游戏**理论上可以直接从浏览器打开**，但使用 HTTP 服务器有以下优势：

1. **✅ 完整功能支持**
   - Web Speech API（单词发音）正常工作
   - 所有浏览器特性都能使用
   - 没有安全限制

2. **✅ 更好的兼容性**
   - 所有浏览器都能正常运行
   - 没有跨域问题
   - 更接近真实部署环境

3. **✅ 更好的调试体验**
   - 浏览器开发者工具正常工作
   - 错误信息更清晰
   - 网络请求可追踪

### 直接打开文件可能遇到的问题

如果你直接双击 `index.html` 打开：

#### 问题 1：只显示静态内容，游戏无法运行
**原因**：浏览器安全策略阻止了 JavaScript 执行

**解决方案**：
- 使用 HTTP 服务器（推荐）
- 或者在浏览器设置中允许本地文件访问（不推荐，有安全风险）

#### 问题 2：语音功能不工作
**原因**：Web Speech API 在某些浏览器的 `file://` 协议下被限制

**解决方案**：
- 使用 HTTP 服务器
- 或使用支持 `file://` 协议的浏览器（如 Firefox）

#### 问题 3：某些浏览器完全阻止
**原因**：Chrome 等浏览器对 `file://` 协议有严格限制

**解决方案**：
- 使用 HTTP 服务器
- 或使用 Firefox 浏览器（限制较少）

## 🔧 如何检查问题

### 打开浏览器控制台
1. 在浏览器中按 `F12` 或右键选择"检查"
2. 切换到 "Console"（控制台）标签
3. 查看是否有错误信息

### 常见错误信息

#### 错误 1：`Game is not defined`
**原因**：game.js 文件未正确加载
**解决**：检查文件路径，使用 HTTP 服务器

#### 错误 2：`Failed to load module`
**原因**：浏览器不支持 ES6 模块或安全限制
**解决**：使用 HTTP 服务器

#### 错误 3：`CORS policy` 错误
**原因**：跨域资源共享限制
**解决**：必须使用 HTTP 服务器
